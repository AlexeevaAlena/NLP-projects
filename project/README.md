# LLM-ассистент для абитуриентов

Интеллектуальный ассистент для абитуриентов, который автоматически отвечает на вопросы о поступлении, используя реальные данные (таблицы и документы) и минимизируя ошибки и галлюцинации.

## Команда
Антикислые

## Цель
Разработать интеллектуального ассистента, который помогает абитуриентам с вопросами о поступлении, предоставляя точные ответы на основе проверенных данных.

## Возможности
- Классификация запросов (FAQ / SQL / RAG / некорректный)
- Поиск по базе знаний (FAISS + эмбеддинги)
- Обработка табличных запросов (генерация SQL, извлечение значений)
- Генерация ответов через LLM
- Корректирующий RAG (проверка на галлюцинации)
- Фильтрация токсичных запросов

## Используемые модели

- **LLM**: Qwen/Qwen2.5-1.5B-Instruct, Mistral
- **Эмбеддинги**: inf1oat/multilingual-e5-base (хорошая поддержка русского языка)
- **Реранкер**: cross-encoder/ms-marco-MiniLM-L-6-v2
- **Генерация SQL**: управление через промпт

## Pipeline

1. Запрос пользователя
2. Классификация (route_node)
3. В зависимости от типа:
   - RAG → поиск (FAISS) → реранкер → LLM → корректирующий RAG → ответ
   - SQL → генерация SQL → запрос к БД → ответ
   - FAQ → поиск в FAISS → ответ
   - Автоответ → прямой ответ

## Корректирующий RAG

Дополнительный этап проверки ответа LLM на соответствие найденным данным:

- Если ответ полезен → вернуть ответ
- Если не полезен → автоответ
- Если галлюцинация → сгенерировать заново
- Если достигнут лимит попыток → остановиться

Приоритет: надежность выше полноты.

## Интерфейс

Реализован на Streamlit:
- Поле ввода вопроса на естественном языке
- Кнопка отправки для запуска pipeline
- Блок с итоговым ответом

Перед обработкой выполняется проверка на токсичность с использованием словарей (ru_abusive_words, ru_curse_words).
